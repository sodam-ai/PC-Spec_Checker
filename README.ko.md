# PC Spec Checker v3.0

**더블클릭 한 번으로 내 PC의 모든 것을 확인하세요.**

설치 필요 없음. 코딩 필요 없음. 전문 지식 필요 없음.
파일 하나만 다운로드해서 더블클릭하면, 내 PC 사양을 바로 확인할 수 있습니다.

> [English version (영어 버전)](./README.md)

---

## 이게 뭔가요?

PC Spec Checker는 `.bat` 파일 **하나**로 내 Windows PC를 스캔해서 보여주는 도구입니다:

- CPU, RAM, GPU, 저장장치가 뭔지
- 디스크 건강 상태는 어떤지
- 어떤 소프트웨어와 개발 도구가 설치되어 있는지
- 내 PC가 개발에 얼마나 적합한지 **100점 만점으로 점수화**

쉽게 말하면, **내 컴퓨터 무료 건강검진**입니다.

---

## 누구를 위한 도구인가요?

- **내 PC 사양이 궁금한 모든 사람** - 무거운 프로그램 설치 없이 바로 확인
- **중고 PC를 사거나 파는 사람** - 스펙 한눈에 정리
- **코딩을 배우기 시작한 초보자** - 내 PC가 개발에 적합한지 확인
- **AI 도구(ChatGPT, Claude, Cursor 등)로 코딩하는 사람** - 개발 환경이 제대로 세팅됐는지 확인
- **IT 지원 담당자** - 원격으로 "이 파일 실행해서 결과 알려주세요"

## 사용법 (3단계)

### 1단계: 다운로드

이 GitHub 페이지에서 초록색 **"Code"** 버튼을 클릭하고, **"Download ZIP"** 을 클릭하세요.

![download](https://img.shields.io/badge/1단계-ZIP_다운로드-brightgreen?style=for-the-badge)

다운로드된 ZIP 파일을 원하는 곳에 압축 해제하세요.

### 2단계: 실행

**`PC-Spec_Checker.bat`** 파일을 찾아서 **더블클릭**하세요.

![run](https://img.shields.io/badge/2단계-더블클릭-blue?style=for-the-badge)

> **Windows 보안 경고가 뜰 수 있습니다.** "Windows가 PC를 보호했습니다" 같은 메시지가 나타나면:
>
> **"추가 정보"** 를 클릭한 후 **"실행"** 을 클릭하세요.
>
> 이 파일은 100% 오픈소스입니다. 메모장으로 열어서 코드를 직접 확인할 수 있습니다.

### 3단계: 확인할 항목 선택

메뉴가 나타납니다. 숫자를 입력하고 Enter를 누르세요.

![menu](https://img.shields.io/badge/3단계-번호_입력-orange?style=for-the-badge)

**추천: `1`을 입력하고 Enter** - 전체 스캔을 실행합니다.

끝입니다. 설치도, 계정도, 인터넷도 필요 없습니다.

## 점수 시스템

PC를 **5가지 항목**으로 평가하여 **100점 만점**으로 채점합니다:

| 항목 | 최대 점수 | 기준 |
|------|----------|------|
| CPU | 20점 | 코어 수 (8코어 이상 = 만점) |
| RAM | 20점 | 총 메모리 (32GB 이상 = 만점) |
| 저장장치 | 15점 | SSD/NVMe 여부 + 남은 용량 |
| GPU | 10점 | 전용 GPU + VRAM 크기 |
| 개발 도구 | 35점 | 필수 도구 설치 여부 |

### 등급

| 점수 | 등급 | 의미 |
|------|------|------|
| 85-100 | **S** | 최고의 개발 머신 |
| 70-84 | **A** | 훌륭한 셋업 |
| 55-69 | **B** | 괜찮음, 개선 여지 있음 |
| 40-54 | **C** | 개선 필요 |
| 0-39 | **D** | 업그레이드 권장 |

점수 확인 후, 구체적으로 **뭘 개선하면 좋은지** 추천도 해줍니다.

### 어떻게 탐지하나요?

각 도구마다 **3단계**로 검색합니다:

1. **시스템 PATH** - 일반적인 방법
2. **알려진 설치 경로** - 도구별로 30개 이상의 설치 경로를 직접 확인
3. **Windows 레지스트리** - GUI 앱은 레지스트리에서 탐지

덕분에 **PATH에 등록하지 않은 도구도 찾아냅니다.**

## 자주 묻는 질문

### 이 파일 실행해도 안전한가요?

네, 안전합니다. 이 파일은:
- 아무것도 **설치하지 않습니다**
- 어떤 설정도 **변경하지 않습니다**
- 어디에도 데이터를 **전송하지 않습니다**
- 인터넷이 **필요 없습니다**
- PC 정보를 **읽기만** 합니다

전체 소스 코드가 `.bat` 파일 안에 있습니다. 메모장으로 열어서 직접 확인할 수 있습니다.

### 관리자 권한이 필요한가요?

아니요. 관리자 권한 없이 실행됩니다. 일부 기능(CPU 온도 등)은 관리자 권한 없이 "N/A"로 표시될 수 있지만, 나머지는 모두 정상 작동합니다.

### Windows 10에서도 되나요?

네. Windows 10과 Windows 11 모두 지원합니다.

### Mac이나 Linux에서도 되나요?

아니요. 이 도구는 Windows 전용입니다. PowerShell과 WMI를 사용합니다.

### 스캔이 느려요

"전체 스캔"은 120개 이상의 도구와 다양한 하드웨어 센서를 확인하기 때문에 30-60초 정도 걸릴 수 있습니다. 개별 메뉴 항목은 훨씬 빠릅니다.

### 설치된 도구가 "not installed"로 나와요

스캐너는 일반적인 설치 경로를 확인합니다. 특수한 위치에 설치했다면 탐지되지 않을 수 있습니다. 해당 도구를 시스템 PATH에 추가하면 해결됩니다.

### "found in PATH"가 뭔가요?

시스템 PATH에서 도구를 찾았지만 버전을 확인할 수 없었다는 뜻입니다. 도구는 설치되어 있고 정상 작동합니다.

## 용어 설명 (완전 초보자용)

이 도구를 사용하면서 모르는 용어가 나올 수 있습니다. 여기서 쉽게 설명합니다:

| 용어 | 쉬운 설명 |
|------|----------|
| **CPU** | 컴퓨터의 두뇌. 모든 계산을 처리합니다. |
| **코어 (Core)** | CPU 안의 작업자 수. 많을수록 동시에 더 많은 일을 합니다. |
| **스레드 (Thread)** | 각 코어가 동시에 처리할 수 있는 작업 줄기. 보통 코어의 2배입니다. |
| **RAM** | 컴퓨터의 단기 기억력. 프로그램을 실행할 때 사용하는 임시 공간입니다. |
| **DDR4 / DDR5** | RAM의 세대. DDR5가 DDR4보다 빠릅니다. |
| **GPU** | 그래픽 카드. 화면 표시, 게임, AI 작업에 사용됩니다. |
| **VRAM** | GPU 전용 메모리. 높을수록 고해상도 작업에 유리합니다. |
| **SSD** | 빠른 저장장치. 프로그램 실행과 부팅이 빠릅니다. |
| **HDD** | 느린 저장장치 (하드디스크). 용량은 크지만 SSD보다 느립니다. |
| **NVMe** | 가장 빠른 SSD 종류. 일반 SSD보다 3-5배 빠릅니다. |
| **PATH** | 컴퓨터가 프로그램을 찾는 주소록. 여기에 등록된 프로그램은 어디서든 실행 가능합니다. |
| **Git** | 코드 변경 이력을 관리하는 도구. 개발자의 "되돌리기" 시스템입니다. |
| **Node.js** | JavaScript를 PC에서 실행하게 해주는 도구. 웹 개발에 필수입니다. |
| **Docker** | 프로그램을 격리된 환경에서 실행하는 도구. 가상 컴퓨터의 가벼운 버전입니다. |
| **WSL** | Windows 안에서 Linux를 실행하는 기능. 개발자에게 유용합니다. |
| **IDE** | 코드를 작성하는 전문 프로그램 (VS Code, IntelliJ 등). 코딩용 워드프로세서입니다. |
| **CLI** | 키보드로 명령어를 입력해서 사용하는 프로그램. 마우스 대신 텍스트로 조작합니다. |
| **UAC** | 프로그램이 시스템을 변경하려 할 때 "허용하시겠습니까?" 물어보는 보안 기능입니다. |
| **방화벽** | 외부에서 내 컴퓨터로 들어오는 접근을 차단하는 보안 기능입니다. |
| **.bat 파일** | Windows에서 명령어를 자동으로 실행하는 스크립트 파일입니다. |

## AI로 코딩하는 분들을 위한 안내

ChatGPT, Claude, Cursor, Copilot 같은 AI 도구를 사용해서 코딩을 하고 있다면, 이 도구가 특히 유용합니다:

### 내 개발 환경이 제대로 세팅됐는지 확인

AI가 "npm install을 실행하세요"라고 했는데 안 되나요? **13번 Dev Tools** 스캔을 실행해서 Node.js와 npm이 설치되어 있는지 확인하세요.

### AI에게 내 환경을 알려주기

전체 스캔 결과를 AI에게 복사해서 보여주면, AI가 내 환경에 맞는 더 정확한 답변을 해줄 수 있습니다.

### 어떤 도구를 설치해야 하는지 확인

**15번 Score** 를 실행하면, 부족한 도구가 뭔지 추천해줍니다.
